# CLAUDE.md

このファイルは、このリポジトリでコードを扱う際のClaude Code (claude.ai/code)へのガイダンスを提供します。

**重要**: このプロジェクトに関する回答は日本語で行ってください。

## プロジェクト概要

これは、サブスクリプション型フリーミアムモバイルアプリケーション向けの汎用的な利用規約とプライバシーポリシー文書を作成するための法的文書プロジェクトです。文書は以下の特徴を持ちます：
- バイリンガル対応（英語と日本語）
- GitHub Pagesでホスティング
- 複数のモバイルアプリプロジェクトで再利用可能
- サブスクリプション収益化に特化（広告なし）

## プロジェクト構造

```
legal/
├── common/                     # 汎用テンプレートとドキュメント
│   └── doc/
│       ├── requirements.md     # 法的文書の詳細要件
│       ├── design.md          # 技術設計とアーキテクチャ
│       └── tasks.md           # 実装チェックリスト
└── [app-name]/                # アプリ固有の実装
    └── GitHub_Pages/          # デプロイ可能な静的サイト
        ├── _config.yml        # Jekyll設定
        ├── index.html         # ランディングページ
        ├── en/                # 英語版
        │   ├── terms.html
        │   └── privacy.html
        ├── ja/                # 日本語版
        │   ├── terms.html
        │   └── privacy.html
        └── assets/            # CSS、JS、画像
```

## 主要要件

### 法的文書の内容
- **利用規約**: ユーザーの義務、サブスクリプションモデル、知的財産権、紛争解決
- **プライバシーポリシー**: データ収集/使用、GDPR/日本の法律への準拠、ユーザーの権利、広告データなし

### 技術要件
- モバイルファーストのレスポンシブデザイン
- 言語切り替え機能
- アンカーリンク付き目次
- 印刷対応レイアウト
- WCAG 2.1 AAアクセシビリティ準拠
- カスタマイズ可能なテンプレート（会社名、連絡先情報）

## よく使うコマンド

### GitHub Pagesへのデプロイ
```bash
# GitHub Pagesにデプロイ
git checkout -b gh-pages
git add .
git commit -m "法的文書を更新"
git push origin gh-pages

# Jekyllローカルテスト（Jekyllがインストールされている場合）
bundle exec jekyll serve
```

### デプロイ後のプロジェクトURL
- `https://[username].github.io/[repository]/[app-name]/GitHub_Pages/`
- 言語別: `/en/terms.html`、`/ja/privacy.html`

## 実装ガイドライン

### 新しいアプリの法的文書を作成する場合
1. `common/`または既存のアプリフォルダからテンプレート構造をコピー
2. 設定内の会社/アプリ固有の変数を更新
3. 広告関連のコンテンツがすべて削除されていることを確認（サブスクリプション専用）
4. HTMLを検証し、レスポンシブデザインをテスト
5. GitHub Pagesにデプロイ

### コンテンツの考慮事項
- 広告、AdMob、広告IDへのすべての参照を削除
- サブスクリプション機能と支払い処理に焦点を当てる
- 明確なデータ保持および削除ポリシーを含める
- 翻訳は文字通りではなく、法的に同等であることを確認

### テストチェックリスト
- 言語切り替えが正しく機能する
- すべてのアンカーリンクが適切にナビゲートする
- モバイル表示が読みやすい
- 印刷レイアウトが機能する
- リンク切れや不足しているリソースがない
